## Описание языка
__Подробное описание языка вы можете посмотреть в файле trans.md__

Токены:
* NAME_TOKEN - последовательность строчных и прописных латинских букв, цифр и символа '_', не начинающаяся с цифры
* WORD_TOKEN - последовательность символов, кроме кавычек
* NUM_TOKEN - десятичное число
* COMP_TOKEN - < | <= | > | >= | && | || | == | !=
* UNARY_OP_TOKEN - ++ | --

Грамматика:
_\<stat>_ ::= _\<for-expr>_ _\<stat>_ | _\<for-expr>_ _\<cmd>_;
_\<for-expr>_ ::= for(_\<init>_;_\<cond>_;_\<action-str>_)
_\<init>_ ::= NAME_TOKEN=NUM_TOKEN | NAME_TOKEN=NAME_TOKEN | ε
_\<cond>_ ::= _\<operand>_ COMP_TOKEN _\<operand>_| ε
_\<operand>_ ::= NAME_TOKEN | NUM_TOKEN
_\<action-str>_ ::=  UNARY_OP_TOKEN NAME_TOKEN | NAME_TOKEN UNARY_OP_TOKEN | ε
_\<cmd>_ ::= print(" WORD_TOKEN ") | ε

## Требования
- Компилятор C++, поддерживающий стандарт C++17
- CMake 3.10+ 
- make, ninja или любую другую систему сборки

## Инструкция по сборке
Создайте папку build в корне проекта, внутри нее вызовите cmake, потом make или другую систему сборки, в зависимости от того, для какой системы вы генерируете скрипт:
```
mkdir build
cd build
cmake ..
make
```
По умолчанию для проекта собираются тесты. Вы можете отключить их сборку, указав:
```
cmake .. -DTESTS=OFF
```

## Использование
Вызовите `./translator --help` для справки. Программа работает в двух режимах:
- Если программе при старте не было передано аргументов командной строки, она читает стандартный вход, пока не встретит конец потока - нажатие Ctrl+D. После этого транслятор исполнит код или выдаст сообщение об ошибке.
- Если программе при старте передан аргумент, то программа попытается открыть файл с названием, переданным в качестве аргумента. Программа прочитает код из файла и исполнит его (или выдаст ошибку)
